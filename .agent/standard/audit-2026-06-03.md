## 二、关键发现（按严重度）

### 🔴 P0-1：本地 hooks 未启用（最大真实缺口）
```bash
$ git config --get core.hooksPath
# 空输出 → Git 默认使用 .git/hooks/，那里是空目录
$ ls .git/hooks/ | grep -v sample
# 空
```
**后果**：`.githooks/pre-commit` 和 `pre-push` 两个文件存在但永不执行。  
RULE-WORKTREE-001 在本地完全靠"开发者自觉"，与文档声称的"机器铁律"严重背离。  
**修复成本**：1 行命令 + 1 行 Makefile target。

### 🔴 P0-2：secret-check 未挂入 pre-commit
`.githooks/pre-commit` 只检查分支名，未调用 `scripts/check_secrets.sh`。  
**后果**：密钥可以在本地 commit 通过，仅在 CI（security.yml）拦截，违反"快速反馈"原则。  
**修复成本**：pre-commit 增加 ~3 行。

### 🟡 P1-1：worktree-guard CI 只在 PR 时触发
`.github/workflows/worktree-guard.yml` 只挂 `pull_request`，**push 到 main 不会触发**。
配合"hooks 未启用 + main 无 branch protection"（待确认），意味着 main 仍可被直接 push。
**修复成本**：workflow 增加 `push: branches: [main]` 触发 + 检查 `github.event.pusher`，5 行。

### 🟢 P2-1：goalcli 单一命名
goal-patch.md v2.0+ 要求机器裁判与执行器使用 `goalcli`。
**裁决更新**：文档、代码、schema、Evidence 和 Makefile 统一改名为 `goalcli`，不再保留历史旧名的并列权威。

### 🟢 P2-2：`.agent/` 90 文件平铺，未按 v1.8 九层分目录
当前结构：所有 yaml/md 平铺在 `.agent/` 根（仅 `rules/`、`evidence/`、`debt/` 等少数子目录）。  
v1.8 建议拆为 L0–L9 九层。  
**裁决**：**不重构**。物理迁移成本 > 收益，且会击穿所有 PR/Issue 引用链。改用"文档分类标签"代替物理迁移。

## 六、Task 3 本地修复复核补充（worker-3，2026-06-04）

本补充只复核 `.agent` control plane、`cmd/goalcli` validator 集成和验证风险；不修改 GitHub Issue，不扩大到物理重构 `.agent/`。结论沿用上文裁决：**不做 L0-L9 物理迁移**，但需要把分类/索引从“文档说明”升级为可被 `goalcli` 验证的机器索引。

### 1. `.agent` 分类 / index 需求

- **当前状态**：`.agent/rules/registry.yaml` 已声明 `status: active|indexed|deprecated`，并在文件头定义 `enforced_by` 语义；`.agent/command-registry.yaml` 与 `.agent/makefile-target-registry.yaml` 仍是 P0 命令/target 索引的主要输入。
- **实际风险**：`.agent/command-registry.yaml:58-114` 和 `.agent/makefile-target-registry.yaml:10-34` 存在重复的 debt/architecture/domain/docs-drift/dependency/testing/implementation/security 条目。只要 validator 仍按字符串 marker 通过，重复索引会制造不稳定分类、重复报告或“最后一条覆盖前一条”的实现差异。
- **文件级建议**：
  - `.agent/command-registry.yaml`：先保留现有内容，下一修复切片增加唯一性校验，再去重并锁定 canonical ordering。
  - `.agent/makefile-target-registry.yaml`：同样增加 target 唯一性校验，报告重复 target 的所有行号。
  - `.agent/rules/README.md` / `.agent/rules/registry.yaml`：明确 `registry.yaml` 是规则分类的 canonical index；若未来生成 compiled index，只能从该 registry 派生，不能并列成为第二权威。

### 2. `cmd/goalcli` validator 集成缺口

- **已集成的正面证据**：`cmd/goalcli/governance.go:341-351` 的 `issue-registry` 已走 `appendIssueRegistryGaps`，符合 `docs/standard/goalcli-cli-contract.md:149` 对 title/command/evidence/status/ID 唯一连续性的语义校验要求。
- **主要缺口**：`runRegistryCheck`（`cmd/goalcli/governance.go:295-315`）仍以 `strings.Contains` 检查 marker；`cli-contract` 和 `command-registry`（`cmd/goalcli/governance.go:330-360`）因此无法发现 YAML 结构损坏、重复 command name、重复 target 或未知字段。
- **doctor 风险**：`runDoctor`（`cmd/goalcli/governance.go:51-80`）只证明必需文件存在，不证明 registry 可解析、无重复、与 Makefile / dispatch / contract 一致。
- **planned command 风险**：`plannedCommandFiles` 与 `plannedCommandSemanticMarkers`（`cmd/goalcli/governance.go:860-1034`）已经阻止部分 dry-run 退化为纯路径检查，但覆盖面只限有 marker 表的命令；其余 planned command 仍可能只验证文件存在。
- **文件级建议**：
  1. 在 `cmd/goalcli/governance.go` 增加 parser 型 registry validator，至少覆盖 `.agent/command-registry.yaml`、`.agent/makefile-target-registry.yaml` 和 `.agent/rules/registry.yaml` 的结构、唯一键、必填字段与未知字段。
  2. 让 `runCommandRegistry`、`runCLIContract`、`runTaskCheck`、`runDoctor` 调用同一 parser validator；`task-check` 不应在 canonical `.agent/command-registry.yaml` 缺失时以 legacy registry warning 通过。
  3. 增加 command-set parity：`commandRegistryRequiredCommands()`、`run` dispatch、`.agent/command-registry.yaml`、`.agent/makefile-target-registry.yaml`、Makefile target 和 CLI contract 必须一一对应；deprecated/legacy 例外需显式列白名单。
  4. 将纯解析/校验逻辑下沉到 `internal/goalcli` 或 `internal/validation`，避免 `cmd/goalcli/governance.go` 继续增长为不可复用的命令层 validator。

### 3. 验证 / 测试风险

- **已有覆盖**：`cmd/goalcli/main_test.go` 已覆盖 governance 命令 surface、usage 和部分 contract marker 缺失；`internal/goalruntime/goalruntime_test.go` 覆盖 authority path 与 final evidence prerequisite；`scripts/check_standard_impact_test.go` 覆盖标准迁移索引的部分回归。
- **缺口**：当前缺少 table-driven mutation fixture 来证明“同一个 `.agent` 损坏”会同时被 CLI validator 与 `internal/goalruntime.Evaluate` 捕获。
- **建议新增测试**：
  - `cmd/goalcli/main_test.go`：重复 command name、重复 make target、缺失 dispatch/contract/make target 的 parity 失败用例。
  - `internal/goalruntime/goalruntime_test.go`：逐个删除 `requiredAuthorityPaths` 中的 canonical authority path，断言 exact gap。
  - `internal/validation/validation_test.go` 或 `internal/goalcli/*_test.go`：损坏 `.agent/rules/registry.yaml` 的 active `enforced_by`，断言 orphan command 失败。
