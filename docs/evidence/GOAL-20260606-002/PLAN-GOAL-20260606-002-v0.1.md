Rollback:

P0-2. Define strict-config-root projection taxonomy.

Acceptance criteria:
- Every root path is classified as one of `platform-native`, `thin-adapter`, `generated-projection`, or `forbidden-legacy`.
- `forbidden-legacy` paths are hard failures.
- Platform-native entries such as GitHub rulesets and CODEOWNERS are not treated as ordinary copied files.

Required gates/evidence:
- New or updated strict config schema tests.
- `GOWORK=off make contracts`
- `GOWORK=off make boundary`
- `GOWORK=off make governance-check`, where available.

Rollback:
- Restore previous strict-config-root profile and keep the new taxonomy behind a disabled config flag until contracts pass.

P0-3. Replace copy-all-then-prune rendering with allowlist materialization.

Acceptance criteria:
- Generator renders only explicit allowlist material.
- Render verification fails on unexpected `.config` source leakage.
- Generated projections have stable checksum and attestation records.

Required gates/evidence:
- `GOWORK=off make render-check`
- `GOWORK=off make integration`
- Projection checksum evidence and downstream render smoke evidence.

Rollback:
- Stop rollout, retain previous generated target, and rerun render verification before retry.

P0-4. Harden PathGuard and platform policy checks.

Acceptance criteria:
- PathGuard blocks symlink, submodule, worktree, case-collision, path traversal, `go:embed`, fixtures, Docker context, and workflow bypasses described in v4/v5.
- Remote GitHub policy audit covers branch protection, rulesets, Actions settings, and CODEOWNERS effective state.
- Local `.github/rulesets/*.json` drift from remote enabled state is release-blocking.

Required gates/evidence:
- `GOWORK=off make pathguard`
- `GOWORK=off make platform-check`
- `GOWORK=off make github-policy-audit`
- Negative-test matrix for every bypass class.

Rollback:
- Fail closed: block release and keep previous manifest/checksum chain.

P0-5. Promote proof-depth and release evidence to typed contracts.

Acceptance criteria:
- Release-blocking claims require typed `proof_depth`; D4 is the minimum for release-blocking gates, and D5 negative-fixture proof is required before release-final promotion.
- Downstream adoption is only claimed with D7 downstream conformance evidence.
- Current release facts are objectized and reconcile tag, manifest, GoalCLI/version constants, changelog/release notes, source provenance, and release decision.
- Evidence ledger is append-only or replayable, includes hash-chain or equivalent integrity evidence, and records command, input, output, exit code, env, digest, time, runner, and policy version.
- Release manifest is a projection of the evidence ledger and records remote policy decision, projection checksums, config profile, and evidence chain.
- Metrics contract changes are objectized as contract artifacts with schema, digest, and proof depth rather than inferred from prose-only docs.
- Task/debt records bind invariant, location, evidence, severity, owner, expiry, remediation, goal, gate, proof depth, and evidence output.

Required gates/evidence:
- `GOWORK=off make release-check`
- `GOWORK=off make release-final-check`
- `goalcli score` where applicable.

Rollback:
- Do not publish release; retain prior release manifest and record a release gap.
