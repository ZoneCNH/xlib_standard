## 1. 底层本质

`baselib-template` 不是运行时基础库，也不是普通脚手架。它的本质是：

> 把 x.go 后续所有基础库的工程规则、目录结构、测试门禁、Release Evidence、生成器不变量和 Agent 执行协议产品化。

它的第一性约束是：

```text
可生成
可验证
可复现
可发布
可迁移
可审计
可被 Agent Teams 执行
```

如果 `baselib-template` 自己不能通过机器门禁证明这些能力，那么后续 `foundationx`、`postgresx`、`redisx`、`kafkax`、`taosx`、`ossx` 都会继承不确定性。

  -> 本地和 CI 生成
  -> CI artifact 上传
  -> Evidence 引用 artifact URL + sha256

```

### PATCH-2：新增 / 修订 Evidence Protocol

目标文件：

```text
docs/standard/evidence-protocol.md
```

#### 可直接提交草稿

```markdown
# Evidence Protocol

## 1. Purpose

Evidence is the auditable proof that a module, generated library, or release gate completed under the current repository facts.

A task, issue, goal, or release MUST NOT be marked as DONE unless it provides verifiable Evidence.

The required completion phrase is:

```text
DONE with evidence:
```

## 2. Release Manifest Lifecycle

`release/manifest/latest.json` is a generated Evidence artifact.

It MUST NOT be committed to source history.

The repository MUST commit:

```text
release/manifest/template.json
```

The repository MUST ignore:

```text
release/manifest/latest.json
```

The generated manifest lifecycle is:

```text
make evidence
  -> generate release/manifest/latest.json locally

make release-check
  -> run ci + integration
  -> generate release/manifest/latest.json with CHECK_STATUS=passed
  -> run release-evidence-check

CI release workflow
  -> generate release/manifest/latest.json
  -> calculate sha256
  -> upload latest.json as CI artifact
  -> expose artifact URL + sha256 in release summary

release-final-check
  -> require gates passed
  -> require manifest consistent with current repository facts
  -> require git workspace clean
```

## 3. Why latest.json Is Not Committed

`latest.json` contains commit-specific, tree-specific, time-specific, and tool-version-specific facts.

Committing it would create one of two failure modes:

1. self-referential commit drift;
2. dirty workspace deadlock after local generation.

Therefore, `latest.json` is generated, verified, uploaded, and referenced, but not committed.

## 4. Required Manifest Fields

The generated manifest MUST include at least:

```text
module
version
commit
tree_sha
source_digest
tracked_file_count
go_version
generated_at
generated_by
tree_state
checks
contracts
dependencies
tools
artifacts
notes
```

## 5. Artifact Reference Rule

Any Evidence reference to `latest.json` MUST use:

```text
artifact_url
sha256
workflow_run_id
commit
tree_sha
generated_at
```

A local file path alone is not sufficient release Evidence.

## 6. Development vs Release Semantics

`make release-check` MAY allow `tree_state=dirty` for local development validation.

`make release-final-check` MUST require:

```text
tree_state=clean
all required checks=passed
manifest matches current repository facts
```

## 7. Evidence DoD

Evidence is valid only if:

1. it was generated by a declared gate;
2. it includes the exact commit or tree state;
3. it includes sha256 for generated artifacts;
4. it contains no raw secret;
5. it can be reproduced or revalidated;
6. it is linked from the task, PR, release, or review record.
```

验收：

```bash
test -f docs/standard/evidence-protocol.md
grep -q "latest.json is a generated Evidence artifact" docs/standard/evidence-protocol.md
grep -q "MUST NOT be committed" docs/standard/evidence-protocol.md
```

#### `scripts/check_docs.sh` 最小实现

```bash
#!/usr/bin/env bash
set -euo pipefail

required_files=(
  "README.md"
  "docs/standard/README.md"
  "docs/standard/xlib-standard.md"
  "docs/standard/repository-roles.md"
  "docs/standard/layering.md"
  "docs/standard/evidence-protocol.md"
  "docs/standard/template-generation-contract.md"
  "docs/standard/dod.md"
  "docs/standard/downstream-compatibility.md"
)

for f in "${required_files[@]}"; do
  if [[ ! -f "$f" ]]; then
    echo "missing required doc: $f" >&2
    exit 1
  fi
done

# Reject unresolved template placeholders in standard docs.
if grep -RInE '\{\{[^}]+\}\}|TODO_TEMPLATE' docs/standard README.md; then
  echo "unresolved template placeholder found in docs" >&2
  exit 1
fi

# Basic markdown link target check for local links.
python3 - <<'PY'
from pathlib import Path
import re
import sys

files = [Path("README.md")] + list(Path("docs").rglob("*.md"))
failed = False

for file in files:
    text = file.read_text(encoding="utf-8")
    for m in re.finditer(r'\[[^\]]+\]\(([^)]+)\)', text):
        target = m.group(1).split("#", 1)[0]
        if not target:
            continue
        if target.startswith(("http://", "https://", "mailto:")):
            continue
        p = (file.parent / target).resolve()
        root = Path(".").resolve()
        try:
            p.relative_to(root)
        except ValueError:
            print(f"{file}: link escapes repo: {target}", file=sys.stderr)
            failed = True
            continue
        if not p.exists():
            print(f"{file}: broken local link: {target}", file=sys.stderr)
            failed = True

if failed:
    sys.exit(1)
PY

echo "docs-check passed"
```

验收：

```bash

make docs-check
```

### PATCH-4：新增 / 修订 Template Generation Contract

目标文件：

```text
docs/standard/template-generation-contract.md
```

#### 可直接提交草稿

```markdown
# Template Generation Contract

## 1. Purpose

`baselib-template` is a generator-backed foundation library template.

The template generator MUST produce downstream repositories that are:

1. Go module independent;
2. free of old template identifiers;
3. contract-consistent;
4. metrics-prefix consistent;
5. release-evidence capable;
6. boundary-clean;
7. safe from accidental overwrite.

## 2. Inputs

The generator MUST require:

```text
--module-name
--module-path
--package-name
--out
```

Example:

```bash
scripts/render_template.sh \
  --module-name foundationx \
  --module-path github.com/ZoneCNH/foundationx \
  --package-name foundationx \
  --out ../foundationx
```

## 3. Output Directory Safety

The output directory MUST NOT already contain a repository or non-empty module.

The generator MUST fail if `--out`:

1. points to an existing non-empty directory;
2. contains `.git`;
3. contains `go.mod`;
4. contains production-looking source files;
5. is equal to the template repository root.

The generator MUST NOT overwrite an existing real repository.

## 4. Copy Exclusions

The generator MUST exclude at least:

```text
.git
.worktree
.omx
node_modules
release/manifest/latest.json
tmp
dist
coverage.out
```

The generator MUST also avoid copying generated CI artifacts and local cache files.

## 5. Replacement Invariants

After rendering, the generated repository MUST satisfy all of the following:

### 5.1 Module Path

`go.mod` MUST contain the requested `--module-path`.

The old template module path MUST NOT remain.

### 5.2 Package Path

`pkg/templatex` MUST be replaced by `pkg/<package-name>`.

Imports referencing `templatex` MUST be replaced.

### 5.3 Identifier Replacement

The generator MUST handle casing variants, including:

```text
templatex
Templatex
TEMPLATEX
{{PACKAGE_NAME}}
{{MODULE_NAME}}
{{MODULE_PATH}}
```

If the generated module requires exported Go identifiers, the generator MUST either:

1. derive stable exported identifiers from package name; or
2. keep neutral names that do not leak template-specific branding.

### 5.4 Metrics Prefix

Metrics names, labels, examples, contract files, docs, tests, and snapshots MUST NOT retain `templatex_` unless explicitly allowlisted.

For `foundationx`, generated metrics MUST use the expected downstream prefix, such as:

```text
foundationx_requests_total
foundationx_request_duration_seconds
foundationx_errors_total
```

The metrics contract MUST match the generated implementation.

### 5.5 Placeholder Elimination

Generated output MUST NOT contain:

```text
{{...}}
TODO_TEMPLATE
__TEMPLATE__
```

unless the occurrence is inside an explicit allowlist file documenting generator examples.

### 5.6 Neutral Path Test

The integration test MUST render at least one downstream module using a neutral non-ZoneCNH path, for example:

```text
example.com/acme/corekit
```

This proves replacement logic does not depend on a specific organization name.

## 6. Generated Repository Gates

Each generated repository MUST pass:

```bash
GOWORK=off go mod tidy
git diff --exit-code go.mod go.sum

GOWORK=off go test ./...
GOWORK=off make contracts
GOWORK=off make boundary
CHECK_STATUS=passed GOWORK=off make evidence
RELEASE_EVIDENCE_REQUIRE_PASSED=1 GOWORK=off make release-evidence-check
```

If `go mod tidy` changes `go.mod` or `go.sum`, the generation gate MUST fail.

## 7. Rendered Template Scanner

The rendered output scanner MUST reject:

```text
github.com/ZoneCNH/baselib-template
pkg/templatex
templatex_ metrics prefix
{{MODULE_NAME}}
{{MODULE_PATH}}
{{PACKAGE_NAME}}
TODO_TEMPLATE
```

The scanner MAY allow `templatex` only inside explicit documentation examples listed in an allowlist.

## 8. Evidence

A successful generation test MUST produce:

```text
rendered module path
rendered package name
go test result
contracts result
boundary result
release evidence result
latest.json sha256
```

This Evidence MUST be referenced from the integration test output or CI artifact.
```

验收：

```bash

grep -q "Metrics Prefix" docs/standard/template-generation-contract.md
grep -q "example.com/acme/corekit" docs/standard/template-generation-contract.md
grep -q "go mod tidy" docs/standard/template-generation-contract.md

#!/usr/bin/env bash
set -euo pipefail

# Existing boundary checks should remain above or below.

# Prevent template reference package from becoming a runtime dependency.
# Allow mentions in docs and tests only if explicitly intended.
if grep -RIn \
  --include='*.go' \
  --exclude='*_test.go' \
  --exclude-dir='.git' \
  --exclude-dir='.worktree' \
  --exclude-dir='.omx' \
  'pkg/templatex\|/templatex\|templatex\.' .; then
  echo "templatex runtime dependency detected outside tests" >&2
  exit 1
fi

# Prevent x.go business dependency.
if grep -RIn \
  --include='*.go' \
  --exclude-dir='.git' \
  --exclude-dir='.worktree' \
  --exclude-dir='.omx' \
  'github.com/bytechainx/x.go\|github.com/ZoneCNH/x.go\|x.go/internal' .; then
  echo "x.go business dependency detected" >&2
  exit 1
fi
```

```markdown
# Layering Standard

## Layers

```text
L-1: baselib-template
L0 : foundationx
L1 : infrastructure libraries
L2 : x.go business adapters
L3 : x.go services
```

## Dependency Direction

Allowed:

```text
L3 -> L2 -> L1 -> L0
L1 -> L0
L0 -> stdlib / minimal dependencies
L-1 -> no runtime dependency from consumers
```

Forbidden:

```text
L0 -> L1
L1 -> L2
L1 -> x.go
L0 -> x.go
baselib-template -> production runtime dependency
```

## Principle

The template defines rules.
The kernel provides reusable runtime primitives.
Infrastructure libraries adapt external systems.
Business adapters own domain semantics.
```

## 15. 最终推荐路径

`baselib-template` 当前方向正确，但完成顺序必须改为：

```text
先修可执行 Gate
再修 Evidence 生命周期
再修生成器不变量
再补标准目录
再生成 foundationx/corekit 验证
最后发布模板库
```

最终判断：

> `baselib-template` 不是写完标准文档就完成；它必须证明自己能稳定生成下游基础库，并且每次生成都能通过独立的 CI、Contract、Boundary、Evidence 和 Release Gate。

真正的完成状态是：

```text
baselib-template
  -> foundationx
  -> corekit
  -> release-check
  -> release-check-extended
  -> release-final-check
  -> DONE with evidence
```

## 17. 结论

你的评审指出的关键问题成立：这份方案真正的风险不是架构方向，而是 `docs-check`、`latest.json`、生成器替换彻底性这三条底层水管。

本 v1.1 的核心变化是：

```text
1. 把 docs-check 提升为 TASK-000
2. 把 latest.json 生命周期写成硬协议
3. 把 generation invariants 写成可执行 Contract
4. 把 Day 1 改成先闭环后填内容
5. 把 templatex 防误用纳入 boundary gate
6. 把 corekit 中性路径纳入 integration invariant
7. 把完成声明绑定到 DONE with evidence
```

这样 `baselib-template` 才能从“标准文档仓库”升级为“可运行的基础库工厂”。
