# ADR-20260602-003: Core Gate 以文档、边界、Evidence 和分数共同定义完成

## 状态

Accepted — 2026-06-02

## 背景

Full Goal Runtime v3.1 的完成条件不只是 MVA 或单一测试通过，而是 release-final-check、release-preflight、score、downstream integration、manifest 和 retrospective patches 的组合 Evidence。

## 决策

Core Gate 必须把以下检查作为同一完成声明的一部分：

- docs-check：标准入口、链接、必需文本和 v3.1 runtime 文件。
- boundary/contracts/integration：模块边界、contract schema 和下游渲染验证。
- release Evidence：`release/manifest/latest.json` 与 `release/manifest/latest.json.sha256`。
- score gate：`goalcli score --min 9.8` 或等价 Makefile target。
- final gate：`GOWORK=off make release-final-check` 与 release-preflight。

## 影响

worker-1 只更新文档和 `.agent` runtime 契约；CLI、Makefile、CI、manifest 与 scoring 实现由其他 worker slice 完成。文档不得把尚未由 gate 证明的实现宣称为已完成。

## Evidence

- `.agent/harness.yaml` 的 final gate 列表。
- `.agent/evidence-protocol.md` 的 completion claim 模板。
- `.agent/traceability-matrix.md` REQ-003/REQ-006/REQ-007/REQ-008 追踪项。
