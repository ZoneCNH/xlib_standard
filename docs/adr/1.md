> 文件名：`baselib_template_completion_goal_plan_v1_0.md`

> 适用项目：x.go 基础库生态
> 文档日期：2026-06-01
> 执行模式：Goal Runtime Prompt v3.1 / Standard Mode
> 目标版本建议：`baselib-template v1.0.0`

---

## 0. 执行结论

`baselib-template` 应作为 x.go 基础库生态的 **第一类：标准 / 模板库** 完成。

它不应该演化为 `foundationx` 的真实运行时代码库，也不应该承载 `postgresx`、`redisx`、`kafkax`、`taosx`、`ossx` 等 L1 基础设施实现。

最终定位固定为：

```text
baselib-template
= 基础库标准
+ 模块模板
+ 生成器
+ Harness Gates
+ Contracts
+ Evidence Protocol
+ Release Manifest
+ Agent Runtime 工件
+ 可编译参考包 templatex
```

当前仓库已经具备良好基础：README 明确它是独立 Go 基础库模板模块；目录中已有 `.agent`、`.github/workflows`、`contracts`、`docs`、`examples`、`internal`、`pkg/templatex`、`release/manifest`、`scripts`、`testkit`；文档中已有 generation、release、spec 等内容；Makefile 已包含 fmt、vet、test、race、lint、integration、security、boundary、contracts、property、fuzz-smoke、golden 等 Gate。

但要真正“完成”，还需要补强标准层入口、仓库角色边界、L0/L1 分层标准、DoD、Evidence Protocol、Issue/PR 执行模板、Retrospective Patch 机制，以及 foundationx / L1 下游兼容性契约。

---

## 1. 问题的底层本质

`baselib-template` 的真实任务不是“提供一个 Go 项目模板”，而是：

> 让所有 x.go 基础库从第一天就带有统一的边界、配置、错误、日志、指标、生命周期、测试、发布、证据链和自进化机制。

因此它的本质是 **基础库生态的工厂**，不是一个普通代码库。

它必须回答这些问题：

```text
一个新的基础库如何生成？
生成后如何证明独立？
如何证明没有依赖 x.go？
如何证明没有隐藏全局客户端？
如何证明配置、错误、健康检查、metrics 与 contract 一致？
如何证明 release manifest 与当前源码一致？
如何证明 foundationx、postgresx、redisx、kafkax、taosx、ossx 都能复用同一套标准？
```

## 2. 不可再拆解的基本真理

### 真理 1：模板库的产物不是代码，而是可复制的工程协议

模板库必须产生：

```text
目录结构
公共 API 形态
Contract
Harness Gate
Release Evidence
CI 工作流
文档骨架
Agent 执行模板
```

而不是只复制几个 Go 文件。

---

### 真理 2：baselib-template 不能混入真实运行时职责

它可以保留 `pkg/templatex` 作为可编译参考实现，但不能把 `xerrors`、`xconfig`、`xlog`、`xmetrics`、`xlifecycle` 的真实实现放进来。

真实 L0 内核应进入：

```text
foundationx
```

真实 L1 基础设施应进入：

```text
postgresx
redisx
kafkax
taosx
ossx
clickhousex
```

---

### 真理 3：生成后必须可证明独立

生成出来的库必须可以在脱离父级 workspace 的情况下运行：

```bash
GOWORK=off make release-check
```

这条是模板库的硬门禁。

---

### 真理 4：没有 Evidence 不能声称完成

完成声明必须使用：

```text
DONE with evidence:
```

并附带：

```text
release/manifest/latest.json
CI run
contract hash
source digest
tool versions
gate results
rendered downstream test
```

---

## 3. 被误认为真理的常见假设

| 常见假设 | 真实情况 |
|---|---|
| 模板库就是复制目录 | 错。它要复制工程协议、Gate、Evidence、Release 机制 |
| baselib-template 可以顺便实现 foundationx | 错。这样会污染边界 |
| 有 README 就算标准完整 | 错。还需要 `docs/standard/*` 作为标准总入口 |
| `make test` 通过就完成 | 错。还需要 contract、boundary、integration、release evidence |
| 生成 `foundationx` 成功就够了 | 不够。还要生成中性路径，如 `example.com/acme/corekit`，验证不依赖特定组织 |
| release manifest 可以提交到源码 | 不建议。`latest.json` 应为生成产物，由 CI artifact 保存 |
| 模板库不需要 Issue/PR 模板 | 错。模板库应把执行协议也模板化 |

## 4. 可以被打破的限制

### 4.1 不需要立即拆成 xlib-standard 与 baselib-template 两个仓库

短期可以让 `baselib-template` 同时承载：

```text
standard
template
generator
gate
evidence
```

但必须在目录内部清晰分层：

```text
docs/standard/      标准入口
docs/                模板模块文档
pkg/templatex/       可编译参考实现
contracts/           JSON schema / metrics contract
scripts/             Gate 和生成器
.agent/              Goal Runtime 工件
release/manifest/    Release manifest 模板
```

### 4.2 可以保留 `pkg/templatex`

但它必须被明确标记为：

```text
reference implementation only
```

不得被业务直接依赖，不得作为 `foundationx` 的真实实现来源。

## 11. 分层标准

`docs/standard/layering.md` 必须固定：

```text
Standard Layer:
  baselib-template

L0 Kernel:
  foundationx/xerrors
  foundationx/xconfig
  foundationx/xlog
  foundationx/xmetrics
  foundationx/xlifecycle

L1 Infrastructure:
  postgresx
  redisx
  kafkax
  taosx
  ossx
  clickhousex

L2 Domain Adapters:
  market_data_storage
  macro_data_storage
  regime_state_store

Business:
  x.go services
```

依赖方向：

```text
Business -> L2 -> L1 -> L0
All generated libraries -> may use template assets
baselib-template -> must not depend on generated libraries
L0 -> must not depend on L1/L2/Business
L1 -> must not depend on L2/Business
```

## 12. Harness Gates 完成要求

### 12.1 Required Gate

必须通过：

```bash
GOWORK=off make ci
GOWORK=off make integration
GOWORK=off make evidence
GOWORK=off make release-evidence-check
GOWORK=off make release-check
```

### 12.2 Final Release Gate

发布前必须通过：

```bash
GOWORK=off make release-final-check
make release-preflight VERSION=v1.0.0
```

### 12.3 Extended Gate

重要变更必须通过：

```bash
GOWORK=off make release-check-extended
```

追加验证：

```text
property
golden
fuzz-smoke
```

### 12.4 Generator Gate

必须验证：

```text
render foundationx
render corekit
check old module path not present
check old pkg/templatex not present
check placeholders not present
go test ./...
make contracts
make boundary
make evidence
make release-evidence-check
```

```text

  - lint:
  - security:
  - contracts:
  - boundary:
  - integration:
  - evidence:
  - release-evidence-check:
  - release-final-check:
- rendered downstream:
  - foundationx:
  - corekit:
- workspace:
  - clean / dirty:
```

禁止：

```text

```
